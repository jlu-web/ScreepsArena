My bot's strategy was giving the creeps specific jobs so that it runs smoothly. I gave my creeps roles like towers, soldiers, scouts, and haulers. The towers would attack enemies first and then heal when there are no enemies near. Soldiers focus on attacking with melee or range and the scouts move toward the enemy flag. The haulers would focus on bringing energy to the tower so that it could keep fighting. I chose this strategy because it is organized and the bot is efficient. 
The part of my code that I am most proud of is how I was able to separate everything into different functions like runSoldiers and runTowers. This made my code easy to read and easy to fix if something went wrong. I also like the hauler system because it was able to supply my tower without getting in my soldiers way.
The most useful JavaScript concepts in this project i used were .filter() and .find(). These helped me give creeps roles and find targets. I also used loops to make them act each tick. Using funtions was important because it made my code easy to understand and organized. 
In the in-class tournament, my bot did pretty well and did not mess up, which helped a lot. It worked the way that I wanted it to and beat almost all the other bots. But it struggled against bots with more defensive strategies. My creeps would split up and try to attack the enemy at the tower and would lose.
If I had more time on this project, I would try to make the bot more advance. I could make it prioritize the weaker enemy and have the soldiers work together better. I would also also try to change the movement so that creeps do not just go straight at the enemy. These changes would make my bot perform better.
